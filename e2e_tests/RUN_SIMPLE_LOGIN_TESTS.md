# Запуск простых тестов аутентификации

## Описание

Создан новый файл `test_simple_login.py` с упрощенными тестами для проверки только входа в систему.

## Тесты

### 1. `test_login_existing_user`
**Основной тест** - проверяет вход существующего пользователя:
- ✅ Открывается экран входа
- ✅ Вводятся email и пароль из конфига
- ✅ Нажимается кнопка входа
- ✅ Проверяется успешный вход

### 2. `test_login_with_wrong_password`
Проверяет, что неверный пароль блокирует вход

### 3. `test_login_with_wrong_email`
Проверяет, что неверный email блокирует вход

### 4. `test_password_visibility_toggle`
Проверяет переключение видимости пароля

## Запуск

### Запустить только основной тест входа:
```bash
cd e2e_tests
python -m pytest tests/test_simple_login.py::TestSimpleLogin::test_login_existing_user -v -s
```

### Запустить все тесты входа:
```bash
cd e2e_tests
python -m pytest tests/test_simple_login.py -v -s
```

### Запустить через bat файл:
```bash
cd e2e_tests
run_tests.bat
# (запустит все тесты, включая простые)
```

## Настройка тестового пользователя

Убедитесь, что в `config.env` указаны правильные данные существующего пользователя:
```env
TEST_USER_EMAIL=ваш@email.com
TEST_USER_PASSWORD=ВашПароль123
```

## Что проверяет основной тест

1. Загрузка экрана входа
2. Ввод email из конфига
3. Ввод пароля из конфига
4. Клик по кнопке входа
5. Проверка успешного входа (исчезновение экрана входа)
6. Появление главного экрана или экрана профиля

## Скриншоты

Тест автоматически создает скриншоты на каждом шаге:
- `1_sign_in_page_opened.png` - экран входа
- `2_email_entered成人.png` - после ввода email
- `3_password_entered.png` - после ввода пароля
- `4_after_login_click.png` - после нажатия входа
- `5_login_successful.png` - успешный вход

## Отладка

Если тест падает:
1. Проверьте скриншоты в `screenshots/`
2. Убедитесь, что тестовый пользователь существует
3. Проверьте, что `testID` добавлены в приложение
4. Пересоберите приложение перед запуском тестов
